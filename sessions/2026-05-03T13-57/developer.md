# ⚙️ [Tech Spec] AI 기반 진단 위젯 구현 명세서 (Closed Loop Architecture)

## 1. 아키텍처 개요: 마이크로 서비스 및 데이터 흐름

본 위젯은 단일 페이지가 아닌, 세 가지 핵심 서비스로 분리된 **마이크로 서비스 아키텍처(MSA)**를 채택합니다. 이는 유연성(주제 변경 용이성)과 확장성(새로운 진단 지표 추가 용이성)을 보장합니다.

### 🔄 데이터 구조적 흐름 (Input → Processing → Output)

| 단계 | 명칭 | 역할 및 기능 | 데이터 형태 |
| :--- | :--- | :--- | :--- |
| **[INPUT]** | **Frontend Widget (External/Blog)** | 외부 블로그에서 사용자가 특정 키워드(예: "AI 규제")를 클릭하거나, 폼에 기본 정보(예: 직무명)를 입력하는 트리거 지점. | `Trigger_Data`: `{ keyword: "AI 규제", source: "External_Blog_A", timestamp: "..." }` |
| **[PROCESSING]** | **Diagnosis Service (Backend)** | 1. `Trigger_Data`를 수신. 2. 핵심 로직(AI/Scoring Algorithm)을 실행하여 점수 및 진단 결과 생성. 3. **가장 중요한 단계:** 사용자의 고유 ID(Session ID)와 진단 결과를 DB에 임시 저장. | `Diagnosis_Result`: `{ score: 75, risk_level: "중상", diagnosis_key: "AI_REGULATION_V2" }` |
| **[OUTPUT]** | **Blog Widget API (Main Blog)** | 1. `Diagnosis_Result`를 수신. 2. **'결과값의 30%만 노출'**하는 템플릿을 사용자에게 제공. 3. "전체 진단 결과 보기" 버튼을 통해 리드 마그넷(LM) 다운로드 페이지로 강제 리다이렉션. | `Teaser_HTML`: (부분 진단 결과 + CTA) |

## 2. 핵심 기술 명세 및 API 연동 방안

### 💻 2.1. API 게이트웨이 (API Gateway)
*   **역할:** 모든 외부 요청을 받는 단일 진입점. 인증, 속도 제한(Rate Limiting), 로깅 및 보안 필터링을 전담.
*   **구현 이유:** 외부에서 들어오는 요청이 악의적 공격이나 과도한 호출로 시스템을 마비시키는 것을 방지하고, 데이터 흐름을 통제합니다.

### 🧠 2.2. 진단 서비스 API (`/api/v1/diagnose`)
*   **Input:** `Trigger_Data` (사용자 입력/키워드)
*   **Process:**
    1.  **스코어링 로직:** (예: 키워드 1 → AI 규제 지표 체크 → 75점 부여). 이 로직은 **외부에 노출되지 않는 블랙박스**여야 합니다.
    2.  **결과 데이터 생성:** 점수, 위험도, 그리고 **결과를 해석하는 전문 용어(진단어)**를 생성합니다.
*   **Output:** `{ success: true, result_key: "AI_REGULATION_V2", teaser_data: { score: 75, summary: "리스크 항목 3가지 중 1개는 즉시 점검 필요" } }`
*   **API 특징:** 이 API는 **데이터를 '생성'**하는 엔진입니다.

### 🔒 2.3. 데이터 수집 및 순환 구조 API (`/api/v1/capture_lead`)
*   **역할:** 위젯을 통해 **'결과값'을 본 사용자**가 다음 단계(LM 다운로드)로 넘어가기 위해 이 API를 호출합니다.
*   **Flow:**
    1.  사용자가 "전체 진단 결과 보기" 버튼 클릭.
    2.  클라이언트가 `diagnosis_key`와 `session_id`를 포함하여 API 호출.
    3.  백엔드는 이 정보를 기반으로 **'미완성된 상태의 리드'**로 인식하고, **이메일/연락처 입력 폼**을 팝업 또는 페이지에 띄웁니다.
    4.  사용자가 이메일 입력 및 동의 시, 최종적으로 **'리드 DB'**에 등록됩니다.
*   **Closed Loop 완성:** 이 과정을 통해 **[외부 유입] → [진단 결과 확인] → [개인 정보 제공] → [LM 다운로드]**의 완벽한 데이터 순환 구조가 완성됩니다.
